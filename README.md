## 📚 เนื้อหาบทเรียน (เพิ่มเติมจากระบบ dlearning.spu.ac.th)

### 📘 [บทที่ 12: การปรับแต่งประสิทธิภาพของระบบบริการ](./chapter12.html)

เรียนรู้เทคนิคการวิเคราะห์และเพิ่มประสิทธิภาพระบบ ตั้งแต่การระบุปัญหาคอขวด (Bottleneck) การใช้เครื่องมือตรวจสอบ (Resource Monitor, Task Manager, Google Lighthouse) ไปจนถึงการปรับแต่งทรัพยากรทั้งในระดับฮาร์ดแวร์และซอฟต์แวร์

**หัวข้อหลัก:**
- ความสำคัญของการปรับแต่งประสิทธิภาพระบบ
- ตัวชี้วัดประสิทธิภาพหลัก (KPIs)
- ปัญหาที่พบได้บ่อย (CPU Bound, Memory Bound, I/O Bound)
- กระบวนการแก้ไขปัญหา 6 ขั้นตอน (พร้อมกรณีศึกษา)
- เครื่องมือตรวจสอบประสิทธิภาพ
- JavaScript Performance Monitor Tool

---

### 📙 [บทที่ 13: การบริหารจัดการปัญหา](./chapter13.html)

เรียนรู้กระบวนการจัดการปัญหาในระดับต่างๆ ตั้งแต่ระดับผู้ใช้จนถึงระดับระบบ พร้อมเทคนิคการวิเคราะห์หาสาเหตุที่แท้จริง (Root Cause Analysis) และการนำเสนอความคืบหน้าโปรเจกต์

**หัวข้อหลัก:**
- การจัดการปัญหาในระดับผู้ใช้
- กระบวนการจัดการปัญหา (Incident Management Process)
- เทคนิคการวิเคราะห์ปัญหา (5 Whys, RCA, Fishbone Diagram)
- การใช้เครื่องมือวิเคราะห์ปัญหา
- การนำเสนอความคืบหน้าโปรเจกต์
- การรับ Feedback และปรับปรุงโปรเจกต์
